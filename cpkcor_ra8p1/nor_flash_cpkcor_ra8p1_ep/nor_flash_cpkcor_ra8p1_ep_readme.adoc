== 驱动文件

在 src 文件夹下，有 `nor_flash.c` 和 `nor_flash.h`，其中包含了 NorFlash 的操作。NorFlash 在 RA8P1 上在读取时可以通过内存映射方式访问，因此头文件中还定义了内存映射的起始地址和 NorFlash 的大小，方便程序内使用

[source,c,subs="verbatim,attributes"]
----
#define NORFLASH_MAP_START_ADDR		0x78000000
#define NORFLASH_MAP_END_ADDR		0x80000000
#define NORFLASH_SIZE				(1024 * 1024 * 64)
#define NORFLASH_SECTOR_SIZE		(1024 * 4)
#define NORFLASH_BLOCK_SIZE			(1024 * 64)
----

对 NorFlash 的写操作由 FSP 提供

[source,c,subs="verbatim,attributes"]
----
fsp_err_t R_OSPI_B_Write(spi_flash_ctrl_t *p_ctrl, uint8_t const *const p_src, uint8_t *const p_dest, uint32_t byte_count);
fsp_err_t R_OSPI_B_Erase(spi_flash_ctrl_t *p_ctrl, uint8_t *const p_device_address, uint32_t byte_count);
----

注意 `R_OSPI_B_Write()` 的参数 `byte_count` 最大为 64。除此之外，有些操作如等待 Flash 内部空闲标志等， FSP 没有提供，因此在 `nor_flash.h` 中提供了对应的 API

[source,c,subs="verbatim,attributes"]
----
uint32_t NorFlash_EraseChip(void);
uint32_t NorFlash_EraseSector(uint32_t sector);
uint32_t NorFlash_WaitOperation(uint32_t timeout);
----

== NorFlash 测试

对 NorFlash 的测试位于 `src/test/test_nor_flash.c` 中，入口函数为 `TestNorFlash()`，该函数会测试读写速度。只需在 hal_entry() 中完成对 NorFlash 的初始化后调用即可

[source,c,subs="verbatim,attributes"]
----
NorFlash_Init();
TestNorFlash(NORFLASH_MAP_START_ADDR, NORFLASH_SIZE);
----

在 `src/test/test_nor_flash.c` 中有一个宏

[source,c,subs="verbatim,attributes"]
----
#ifndef TEST_NOR_FLASH_EN_FULL_CHECK
#define TEST_NOR_FLASH_EN_FULL_CHECK	0
#endif
----

该宏在工程中关闭，若打开，会对全片进行读写校验，但要注意，全片的读写校验耗时约至少五分钟
