== 运行

在 e^2^ studio 中导入工程，生成并编译，或直接烧录本文件夹下的 audio_codec.hex。底板上的 SW302 的 3 和 4 开关拨到 ON，其它拨到 OFF。打开一个串口调试工具，波特率 2000000，数据位 8bit，停止位 1bit，无奇偶校验。注意不要使用如 MobaXterm 这类的串口终端，而是使用如 ATK-XCOM 这类的串口调试工具。程序没有实现 Linux Shell 下的行规程，因此在终端下的输入不会回显。程序烧录后，串口会输出以下信息：

== 驱动代码

=== FSP

ES8311 使用 I2C 进行寄存器配置，使用 I2S 与 RA8P1 传输音频数据，以及一个定时器提供音频时钟 MCLK。因此在 configuration.xml 中添加三个 stack

=== 驱动文件

ES8311 驱动文件位于 src 目录下，es8311.c 和 es8311.h。关于 I2C 的操作在 iic.c 和 iic.h 中。在程序入口 hal_entry() 中先进行初始化

[source,c,subs="verbatim,attributes"]
----
IIC_Init();
ES8311_Init();
----

随后可使用 es8311.h 中的 API 发送和读取音频数据

[source,c,subs="verbatim,attributes"]
----
/* 读取音频数据 */
uint32_t ES8311_Receive(void *data, uint32_t length);
/* 发送音频数据 */
uint32_t ES8311_Transmit(void *data, uint32_t length);
/* 设置音量 */
void ES8311_SetVolume(uint8_t percent);
----
