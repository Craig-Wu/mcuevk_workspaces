== CPK-RA8T2样例程序

本目录下除了readme文件不放置其他文件。

CPK-RA8P1由两个板子组合而成
核心板：CPKNET-RA8T2
扩展板：CPKEXP-ECSMCB

cpk_ra8t2 目录下存放CPK-RA8T2开发板的样例程序, 非此硬件平台的样例程序请放到其他对应目录。
请按规则新建样例代码目录。

在样例程序的readme中也要注明对应使用的CPK板型号（包括核心板和扩展板）


=== 样例程序基本框架

. 样例程序的通用考虑
    .. 使用CPK-RA8x1相同的目录结构
    .. 基于需求文件里注明的FSP版本, 使用CPK-RA8x2核心板配套的BSP (支持FSP 6.2.0或更高版本)
    .. 基于BSP修改的管脚配置，请填入Excel表格，方便客户查询和参考
    .. 使用LLVM编译器，如果可能，再加入AC6编译器 （LLVM 18.1.3， AC6.23）
    .. 提供基于RTT-Viewer和UART console（SCI9_B）的终端控制功能，可以使用编译选项选择使用哪个
        ... 这是为了方便没有JLink的客户和适应最终客户板子上只有串口的情形
    .. 建议加入perf_counter（由ARM提供）作为性能评测工具。
        ... perf_counter支持使用PMU和Tick时钟 （Release版本可以用PMU；调试时某些调试动作可能会影响PMU计数值，可以使用Tick时钟）
        ... 已经在CPK-RA8x1上porting好了perf-counter，支持LLVM编译器和AC6编译器
        ... perf-counter自带CoreMark，可以简单验证项目配置的的优化程度（编译选项、段地址设定等）


. 样例程序需要对所实现的功能进行性能评测
    .. 样例程序至少需要两个编译配置，“Debug” 和 “Release”
    .. Release配置完全偏向于性能，禁用调试相关功能，并且开最大性能优化
    .. 如果可能，再使用AC6编译器做一版评测

. 性能评测的基本要求
    .. 单一功能的性能评测缺省使用使用CPU0，并使用I-TCM和D-TCM （例如 Coremark）
        ... 如果要评测CPU1，会在需求中注明
    .. 复杂功能的评测需要设置合理的存储器用例，并尽量复现实际应用场景
        ... 如评测LCD显示性能，则需要在LCD刷新的同时，向LCD Buffer写入数据，模拟真实用例
        ... LCD buffer需要放到non-cacheable的地址空间
        ... 测试16位数据写入(RGB565输出)和32位数据写入（RGB888输出）
    .. 所有相关的时钟频率均设置均为最大值
    .. 如果使用到了TCM以外的区域，指令Cache和数据Cache缺省打开，但可能需要按照性能评测要求进行设置

. 样例程序的交付
    .. 和CPK-RA8x1样例程序交付要求一致的代码和文档
    .. BSP管脚配置的Excel文件，重要的设置请在项目文档中特别注明（如某些端口需要开漏输出，需要设置片内上拉，驱动能力有特别要求，等等）- 即让用户知道为什么这么配置。Excel文件放到项目根目录下。
    .. Release配置下编译的HEX文件（性能最优），可以直接烧录运行。放在项目根目录下

=== 所有正式发布的样例程序基于MP版本的CPK硬件，CPK核心板上的RA8P1/T2 MCU为ES3版本或更新的版本

    * 如果必要，可以使用早期版本的CPK进行部分开发，但最终验证请使用MP版本的CPK。
    * 量产MP版本的核心板上有激光打印的核心板型号
        ** AE手上有几个未打印型号的MP版CPKNET-RA8T2和CPKHMI-RA8P1，板子背面J201通孔附近有飞线。
