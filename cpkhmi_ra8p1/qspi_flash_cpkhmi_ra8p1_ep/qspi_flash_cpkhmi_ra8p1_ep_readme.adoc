== 驱动文件

在 src 文件夹下，有 `w25qxx.c` 和 `w25qxx.h`，其中包含了 QSPI Flash 的操作。QSPI Flash 在 RA8P1 上在读取时可以通过内存映射方式访问，因此头文件中还定义了内存映射的起始地址和 QSPI Flash 的大小，方便程序内使用

[source,c,subs="verbatim,attributes"]
----
#define W25QXX_MAP_START_ADDR	0x80000000
#define W25QXX_MAP_END_ADDR		0x90000000
#define W25QXX_SIZE				(1024 * 1024 * 32)
#define W25QXX_SECTOR_SIZE		(1024 * 4)
#define W25QXX_BLOCK_SIZE		(1024 * 64)
----

驱动文件中对 QSPI Flash 提供的 API 如下，主要提供擦除和写入操作，读取操作可以直接通过内存映射的方式完成

[source,c,subs="verbatim,attributes"]
----
uint32_t W25QXX_EraseChip(void);
uint32_t W25QXX_EraseSector(uint32_t sector);
uint32_t W25QXX_GetID(uint8_t *manufacturer_id, uint8_t *device_id, uint64_t *unique_id);
uint32_t W25QXX_Init(void);
uint32_t W25QXX_WaitOperation(uint32_t timeout);
uint32_t W25QXX_Write(uint8_t * const src, uint8_t * const dest, uint32_t length, uint8_t erase);
uint32_t W25QXX_WriteEnable(uint32_t timeout);
----

== QSPI Flash 测试

对 QSPI Flash 的测试位于 `src/test/test_qspi_flash.c` 中，入口函数为 `TestQSPIFlash()`，该函数会测试读写速度。只需在 hal_entry() 中完成对 QSPI Flash 的初始化后调用即可

[source,c,subs="verbatim,attributes"]
----
W25QXX_Init();
TestQSPIFlash(W25QXX_MAP_START_ADDR, W25QXX_SIZE);
----

在 `src/test/test_qspi_flash.c` 中有一个宏

[source,c,subs="verbatim,attributes"]
----
#ifndef TEST_QSPI_FLASH_EN_FULL_CHECK
#define TEST_QSPI_FLASH_EN_FULL_CHECK	0
#endif
----

该宏在工程中关闭，若打开，会对全片进行读写校验，但要注意，全片的读写校验耗时约至少十分钟
